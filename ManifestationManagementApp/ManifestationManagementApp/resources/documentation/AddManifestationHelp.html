Adding manifestation

When adding manifestation, user can enter following data:

  • Id - Value of manifestation identifier (id) must be unique and can be manually entered. Also, there is an option for automatic generation of id by selecting "Auto-generated Id" field. If id is entered manually there is length constraint: from 2 to 60 characters.
  • Name - Represents manifestation name. It has length constraint: from 2 to 60 characters.
  • Type - Allows user to choose manifestation type relying on existing manifestation types in system. Manifestation type must be chosen.
  • Smoking allowed - Represents if smoking is allowed on manifestation.
  • People with special needs support - Represents if manifestation is suited for people with special needs.
  • Date - Selection of date when manifestation is being held.
  • Environment - Represents if manifestation is occuring on open area or in closed space.
  • Alcohol consumption - Represents if alcohol consumption is allowed at manifestation, and if it is allowed, is it allowed to buy it by yourself and use on event.
  • Price category - Represents manifestation price range.
  • Label - Allows user to add labels for current manifestation. At least one label must be added.
  • Icon - Choose icon for manifestation. If not chosen, manifestation type icon will be used.
  • Description - More detailed description about manifestation. It has length constraint with minimum 2 characters.

By clicking on button "Add manifestation" user adds manifestation to system. In case of clicking on button "Cancel" adding manifestation is cancelled and user is redirected to window that shows current manifestations in system.
